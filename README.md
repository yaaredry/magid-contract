## איך זה עובד

`contract.pdf` הוא PDF עם שדות טופס אמיתיים (AcroForm text fields) שנוספו בעורך PDF חיצוני – לא תמונה שטוחה ולא DOCX.  
בעת ההורדה, `pdf-lib` (בדפדפן) קורא את המיקום המדויק (Rect) של כל שדה, ומצייר את הערך שהמשתמש הזין ישירות בעמוד באותו מיקום, עם התאמת גודל גופן אוטומטית כדי שהטקסט לא ייחתך. לאחר מכן השדות המקוריים מוסרים כך שנשאר PDF סופי ונקי (לא אינטראקטיבי).

מכיוון שהטקסט מצויר ישירות (ולא דרך docx/מרת HTML), אין תלות בפונטים של מערכת ההפעלה ואין שונות בין Word/Pages/Mac/Windows.

## מיפוי השדות

`PDF_FIELD_MAP` בתוך `index.html` ממפה בין מפתחות ה-wizard (למשל `name`, `phone`) לבין שמות השדות בפועל בקובץ ה-PDF (למשל `text_11time`).  
נכון להיום, לשלושה שדות אין קופסת מילוי אמיתית בתבנית (רק מקף מקושט): `fax`, `type-of-rooms`, `more-dates`. יש להוסיף עבורם שדה טופס אמיתי בקובץ המקור ואז לעדכן את המיפוי.

## עדכון החוזה

אם תבנית החוזה משתנה (טקסט, שדות נוספים וכו'):
1. עדכן את קובץ ה-PDF במקור (בעורך שבו נוספו השדות), והוסף/הזז שדות טופס בהתאם.
2. ייצא ושמור בשם `contract.pdf`.
3. עדכן את `PDF_FIELD_MAP` ב-`index.html` אם שמות השדות השתנו.
4. הרץ `npm test` – הבדיקות יזהו שדות חסרים או שמות שלא תואמים.
